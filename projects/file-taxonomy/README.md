**Owner:** Brian Jensen
**Status:** Draft v2 - 2026-05-19 - Phase 0 foundation complete; Phase 1 (SOP authoring) next.

## Purpose

Establish a documented file taxonomy and organization SOP for Red Door Collaborative. Replace emergent, undocumented file storage with a function-based, metadata-aware system that a new hire can adopt on day one and existing staff (Chris Telfer in particular) can continue to use without disorientation.

## Locked Decisions

See `plan.md` "Locked Decisions" section. Highlights:

- SharePoint canonical storage; Loop org map
- Taxonomic naming: `[Client/RDC]-[FYXX or YYYY]-[PO# or Category]-[FileDescriptor]-[Rev].ext`
- R1/R2/FINAL is a governance workflow, not just a suffix
- Microsoft consolidation via private channels under one master team
- Internal RDC content never archives; client work archives on client's FY rollover
